Summary

파일 임포트의 LLM 교정을 STT 뒤에 숨기는 병렬 파이프라인(Phase 1)을 구현·리뷰·머지했다. wall-clock의 절반 이상을 차지하던 교정 대기가 사라져 임포트가 STT 시간 수준으로 수렴한다. 다음 갈래는 세 개: ① Phase 2 배치화(조건부 보류), ② 화자분리 도입(오디오 보존 정책 결정 대기), ③ Silero VAD main 승격(검증된 조합 단위). 실행 중인 앱은 이전 빌드라 재시작해야 적용된다.

1. 구조 변화 — 무엇이 빨라졌나

기존에는 청크마다 네트워크 교정(3~8초)이 STT를 막았다. 이제 STT(ANE-bound)는 직렬을 유지하고, 네트워크 대기인 교정만 동시 상한 3의 백그라운드 task로 겹친다.

  • 순서 보존: segment는 추가 시점 index로 고정 — 교정 완료 순서와 무관
  • fail-soft: 교정 실패·nil·취소 → 해당 청크만 원문 유지
  • 교정 문맥: 직전 교정문 5개 → 직전 원문 5개 (직렬 의존성 제거, 트레이드오프 문서화)
  • 취소 전파: drain 중 취소 시 slot 대기자까지 즉시 wake (리뷰 M1 반영)
