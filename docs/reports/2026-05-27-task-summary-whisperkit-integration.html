주요 기술 결정

Swift 6 @preconcurrency import WhisperKit

WhisperKit은 Sendable을 conform하지 않으므로, @MainActor 컨텍스트에서 nonisolated 메서드 호출 시 SendingRisksDataRace 에러가 발생합니다. Swift 6 공식 마이그레이션 경로인 @preconcurrency import WhisperKit을 적용하면 모듈 타입이 암묵적 @unchecked Sendable로 처리되어 에러가 해소됩니다.

설계 @MainActor STTService — @ModelActor 제거

이전 코드는 whisper.cpp OpaquePointer를 보호하기 위해 커스텀 global actor(@ModelActor)를 사용했습니다. WhisperKit은 CoreML 내부에서 스레딩을 직접 관리하므로 STTService를 @MainActor로 격리하고 await으로 서스펜드하는 것만으로 UI 블록 없이 안전하게 동작합니다.

테스트 VAD 캘리브레이션 — 저진폭 캘리브 → 고진폭 스피치

VADProcessor는 처음 10프레임을 noise floor 기준으로 사용합니다. 같은 진폭의 신호를 즉시 피드하면 신호 자체가 noise floor가 되어 아무것도 검출되지 않습니다.

올바른 테스트 설계 
// 캘리브레이션: 저진폭(0.05) → noiseFloor ≈ -29 dB, threshold ≈ -23 dB
let calibSamples = makeSineWave(seconds: 2, hz: 440, amplitude: 0.05)

// 스피치: 고진폭(0.8) → -5 dB > threshold(-23 dB) → 검출됨
// 80000 샘플(5s) 도달 시 maxSamples flush 자동 실행
let speechSamples = makeSineWave(seconds: 6, hz: 440, amplitude: 0.8)
빌드 에러 해결 이력 (3건)

1. TranscriptionResult naming conflict

typealias WKResult = WhisperKit.TranscriptionResult — 컴파일러가 WhisperKit을 모듈이 아닌 클래스 이름으로 해석.

// before (error)
typealias WKResult = WhisperKit.TranscriptionResult
let wkResults: [WKResult] = try await pipe.transcribe(...)

// after (OK) — 타입 추론으로 WhisperKit 모듈의 TranscriptionResult 획득
let wkResults = try await pipe.transcribe(...)

2. @Sendable closure 에러

@MainActor-isolated self를 캡처한 클로저를 nonisolated 메서드에 전달.

// before
progressCallback: { [weak self] progress in ... }
// after
progressCallback: { @Sendable [weak self] progress in ... }

3. pipe sending 에러 (region isolation)

Swift 6 region isolation 분석이 @MainActor 컨텍스트에서 읽힌 pipe를 main actor region으로 태깅.

// before
import WhisperKit
// after
@preconcurrency import WhisperKit
