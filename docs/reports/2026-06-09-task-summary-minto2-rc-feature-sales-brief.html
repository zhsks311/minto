Summary

Minto2 RC는 단순 녹음 앱이 아니라, 회의 입력부터 정리, 검색 답변, 외부 공유까지 한 흐름으로 묶는 업무 결과물 생산 도구로 확장됐다. 데모에서는 "회의를 넣으면 바로 써먹을 수 있는 기록이 나온다"가 핵심 메시지다.

한 문장 영업 멘트

이 릴리즈의 판매 포인트는 명확하다. Minto2는 회의가 끝난 뒤 사람이 다시 뒤지는 시간을 줄이고, 로컬과 클라우드 AI를 상황에 맞게 골라 쓰면서, 필요한 근거와 내보내기까지 한 화면에서 이어준다.

고객에게 팔 수 있는 기능

READY 입력 선택지가 넓어졌다

마이크만 받는 회의에서 벗어나 시스템과 마이크+시스템 입력을 선택할 수 있다. 화상회의 상대방 소리를 기록 흐름에 태울 준비가 됐다.

Evidence: SystemAudioLiveTests, AudioInputModeTests
READY 파일을 넣으면 회의록이 된다

파일 가져오기는 분석, 전사, 교정, 요약, 저장을 pipeline으로 돈다. 사용자는 버튼 하나로 녹음 파일을 회의 기록으로 전환한다.

Evidence: MeetingFileImportUseCaseTests, rendered file import QA
READY 검색은 답변까지 간다

저장된 회의를 검색하고, 상위 근거 chunk만 AI에 넘겨 답변과 citation을 함께 만든다. 그냥 검색 결과 목록이 아니라 "질문에 대한 답"을 제공한다.

Evidence: MeetingSearchAnswerServiceTests, local LLM UI QA
READY 로컬 LLM도 제품 흐름에 연결됐다

Ollama와 OpenAI-compatible endpoint를 설정 화면에서 다룰 수 있고, 교정·요약·검색 답변 use case별 payload가 검증됐다. API key가 필요 없는 로컬 모드가 제품 옵션이 됐다.

Evidence: LLMProviderTests, benchmark runner
READY 연결 실패가 덜 당황스럽다

Notion과 Confluence가 인증 실패 상태에 들어가면 일반 빈 결과 대신 재연결 안내를 먼저 보여준다. Confluence 내보내기 실패도 Settings handoff로 이어진다.

Evidence: RelatedInfoTests, IntegrationReconnectStateTests
READY 개발 QA용 secret store가 생겼다

MINTO_DEV_SECRET_STORE=file로 Keychain 대신 격리된 파일 저장소를 쓸 수 있다. 저장·로드·삭제 QA를 반복해도 실제 계정 상태를 덜 건드린다.

Evidence: SecretStoreTests, Settings UI save/load/delete QA

작은 글씨: 몰래 말하는 실제 한계

데모는 된다. 다만 기본값으로 밀기 전에는 아래를 숨기면 안 된다. 로컬 LLM은 앱 후처리 기준 품질 gate를 통과했지만 평균 28.813초다. 특히 summary와 answer case는 평균 35초대라 "빠르다"가 아니라 "로컬에서도 된다"로 말해야 한다.

LIMIT Local LLM 기본값 승격은 아직 이르다

llama3.1:8b는 app-postprocessed quality gate를 통과했지만 latency 정책이 없다. 기본값으로 올리려면 허용 latency, 반복 안정성, 모델 설치 UX를 먼저 정해야 한다.

LIMIT System audio는 권한 거부 상태 QA가 남았다

현재 환경은 이미 화면/시스템 오디오 권한이 허용돼 있었다. 권한 없음 warning, start disabled, 시스템 설정 열기, 권한 허용 후 readiness 갱신은 실제 macOS 권한 상태를 바꿔 확인해야 한다.

LIMIT 실제 화상회의 audio 품질은 별도 측정 대상이다

별도 프로세스 system output callback은 opt-in live test로 확인했다. 하지만 Google Meet/Zoom 같은 실제 앱 출력, 마이크 재유입 echo, 장시간 drift는 아직 제품 품질 판단 데이터가 아니다.

LIMIT Confluence rendered reconnect/delete는 signed app 기준 확인이 필요하다

서비스 레벨 회귀 테스트와 file-store token 삭제는 확인됐다. 다만 direct SwiftPM executable에서는 seeded 상태 렌더가 제한돼 signed app bundle 환경의 실제 UI QA가 남아 있다.

LIMIT Keychain prompt count는 실제 macOS 대화상자 기준 검증이 남았다

상태 조회 경로가 token 원문을 읽지 않는 계약은 자동 테스트로 고정했다. 하지만 반복 Settings 진입 때 실제 prompt가 몇 번 뜨는지는 사용자 권한 환경에서 직접 세야 한다.
