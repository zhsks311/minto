2. 어떻게 동작하나

라이브(빠름·임시) → 저장 시 정확 AI로 재정리 → 확정. 같은 일을 두 번 해 "실시간"과 "정확" 둘 다 잡는다.

3. 핵심 동작 5가지

1실시간 임시 라벨

녹음 중 0.1초마다 "지금 누가 말하나"를 추측해 화면에 표시(최대 10명). 빠른 대신 라벨이 조금 흔들릴 수 있고, 그건 저장 시 고쳐진다.

2앵커 — 아는 사람 먼저 고정 ⚓

"내 마이크 = 나"는 100% 알고 시작. (이후 단계) 전에 등록한 사람은 목소리 지문으로 자동 인식. 맨바닥 추측보다 정확.

3저장 시 정확하게 확정

회의가 끝나면 녹음 전체를 정확한 AI(VBx)로 다시 분석해 최종 라벨로 자동 교체. 저장된 회의록은 DER 12% 수준 정확도.

4사용자 편집 보존

녹음 중 사용자가 직접 고친 화자 라벨은 저장 시 재정리가 덮어쓰지 않는다. 사람의 수정이 항상 우선.

5문제 시 자동 후퇴 (fail-soft)

라이브 화자분리가 에러·과부하면 앱이 알아서 기존 "나/상대" 채널 라벨로 강등. 녹음·받아쓰기는 절대 안 멈춘다.

5. 솔직한 열린 질문

기술적 "고장" 위험은 낮습니다(성능 실측 통과, 문제 시 자동 후퇴). 진짜 판단 포인트는 "라이브의 임시 라벨이 (한국어에서) 보여줄 만큼 쓸 만한가"입니다 — 라이브 AI는 부정확할 수 있고(정확도는 저장본이 책임짐), 너무 흔들리면 라이브는 "나/상대"만 보여주는 가벼운 모드로 둘 수도 있습니다. 큰 구현 전에 짧은 라이브 스모크(실제로 띄워 라벨 흔들림 확인)로 가늠하는 게 합리적입니다.
