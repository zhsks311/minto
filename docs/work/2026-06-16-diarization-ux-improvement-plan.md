# 화자분리 UX·정확도 개선 계획 (3종)

작성일: 2026-06-16
배경: Phase 2 측정으로 확정 — 자동 화자 수 추정은 본질적으로 불안정(K 추정 문제), exactSpeakerCount가 견고. 자동 폴백은 기본 파라미터라 under-count(4인→2). 음질은 병목 아님(exactN=4 → 균형 분할 [108,76,32,27] 확인). 영리 업체는 채널분리·보이스프린트·메타데이터힌트+사후교정 조합으로 해결.

목표: 단일 마이크 믹스 오디오라는 가장 어려운 조건에서, 현실적 3종 개선으로 사용자 체감 정확도를 올린다.

## Phase 1 — 인원 입력 안내 강화 (S, 즉시)

문제: "예상 참석 인원"이 선택값이라 비우면 자동 추정(under-count)이 도는데 안내가 없어, 사용자가 2명 결과를 맹신.

작업:
1. `FileImportSetupSheet` 인원 칸 아래 힌트: "비우면 화자 수를 자동 추정해요. 인원을 알면 입력하는 게 더 정확합니다."
2. 자동 추정으로 라벨된 회의에 표식: 전사 화자 블록에 "자동 판단됨 · 정확하지 않으면 인원을 넣어 다시 가져오기" 안내(MeetingRecord에 작은 플래그 또는 휴리스틱). 최소 구현은 시트 힌트만으로 시작 가능.

검증: 빌드 + 앱에서 시트 힌트 노출 확인. 회귀 없음.
브랜치: feat/speaker-label-edit-ui (기존 교정 UX 브랜치에 합류) 또는 신규.

## Phase 2 — 사후 화자 교정 확장: 분리(split)/구간 재배정 (M)

문제: 현재 교정 UX는 이름변경·병합만. 자동이 under-count(2명)하면 한 화자에 여러 사람이 뭉쳐도 **쪼갤 수단이 없어** 재임포트가 유일.

설계 결정 필요(아래 옵션 중 택1):
- (a) **구간 단위 재배정**(권장): 전사 각 구간에 화자 드롭다운 → 사용자가 틀린 구간을 다른/새 화자로 옮김. 가장 단순·정확·결정적. 기존 `SpeakerLabelEditing`에 "새 화자 생성 + 단일 구간 speaker 변경" 추가.
- (b) 시점 기준 split: 한 화자를 특정 시각 전/후로 둘로. 단순하나 현실 대화에 안 맞음.
- (c) 해당 화자 구간만 K=2로 재-diarize: 정확하나 오디오 재보관 필요(현재 audioFileName=null이라 불가) → 제외.

추천: (a). 순수 함수(구간 재배정·새 라벨 생성) 분리 + 단위 테스트. 기존 merge/rename와 일관.
검증: 빌드 + 단위 테스트 + 앱 QA(시드 회의로 재배정→저장 확인).
브랜치: feat/speaker-label-edit-ui 확장.

## Phase 3 — 반복 참석자 보이스프린트 (L, 설계 먼저)

문제: 팀 정기회의는 같은 사람들이 반복. 한 번 등록하면 이름으로 식별 → 비지도 군집화를 지도 식별로 전환(업체 방식).

제약/발견:
- 오프라인 VBx 매니저엔 enrollment API 없음. 스트리밍(Sortformer/LS-EEND)엔 `enrollSpeaker` 있으나 Sortformer는 4 캡·느림.
- FluidAudio `EmbeddingExtractor` + `PLDATransform` 노출됨 → **직접 구축 가능**: 등록 샘플에서 임베딩 추출·저장 → 임포트 오디오의 화자 클러스터 임베딩을 등록 템플릿과 PLDA/cosine 매칭 → 이름 부여.

구성요소(신규):
1. 등록 UX: 사람별 음성 샘플(짧은 녹음/파일) → 임베딩 추출 → 이름과 함께 저장(보이스프린트 스토어).
2. 식별 통합: diarization 후 각 화자 클러스터 대표 임베딩을 등록 템플릿과 매칭(임계값 이상이면 이름, 미만이면 "화자 N").
3. 프라이버시: 음성 임베딩 로컬 저장, 삭제 수단.

→ **설계 스파이크 먼저**(임베딩 추출/매칭 PoC를 평가 하니스에서 검증) 후 제품 배선. 단일 세션에 무리 — 별도 에픽.

**PoC 결과 (2026-06-16, GREEN, 커밋 e3c77fa, 브랜치 feat/voiceprint-poc):**
- `VoiceprintFeasibilityTests` + `diarizeWithEmbeddings`(청크별 (speakerId, embedding) 추출, exposeChunkEmbeddings).
- 4인 파일: **intraCosine=0.88, interCosine=0.48, gap=0.40** → 화자 임베딩이 강하게 변별적. 등록 템플릿이 자기 화자엔 ~0.88, 남엔 ~0.48로 매칭 → 임계값 ~0.65로 식별 가능.
- **결론: 보이스프린트 식별 실현 가능.** 남은 것 = 제품 구현(등록 UX·보이스프린트 스토어·식별 통합·프라이버시). 별도 세션 권장.
