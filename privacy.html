Data Boundary

무엇이 Mac 안에 있고, 무엇이 나갈 수 있는지.

Minto2는 “로컬”이라는 말 하나로 모든 처리를 뭉뚱그리지 않습니다. 음성 전사, 텍스트 요약, 검색 답변, 토큰 저장은 서로 다른 경계를 갖습니다.

안

음성 전사는 로컬

회의 음성은 WhisperKit 기반 로컬 STT 흐름에서 처리됩니다. 전사를 위해 회의 음성 자체를 서버로 올리는 방향이 아닙니다.

선

LLM은 선택

교정, 요약, 답변 생성에서 클라우드 provider를 고르면 텍스트가 외부로 나갈 수 있습니다. 로컬 LLM을 고르면 이 경계를 줄일 수 있습니다.

잠

비밀값은 Keychain

API token 같은 비밀값은 기본적으로 macOS Keychain에 저장합니다. 개발용 file 기반 저장은 opt-in 경로입니다.

데이터별 처리 경계

회의 음성마이크, 시스템 사운드, 혼합 입력Mac 안에서 전사
원본 전사회의 중 쌓이는 텍스트와 저장된 전사요약·교정 선택 시 전송 가능
검색 답변저장된 회의에서 찾은 관련 근거 조각사용자가 답변 생성을 요청할 때만
참고 문서로컬 파일, Notion, Confluence 문맥요약·발췌·용어 중심 활용
토큰과 키LLM provider, Confluence 같은 연동 비밀값Keychain 저장
로그오류 진단과 상태 추적용 이벤트프롬프트·토큰·전사 원문 제외

클라우드 LLM을 선택하는 순간에도 전체 회의를 무조건 보내는 구조가 아닙니다. 기능별로 필요한 텍스트, 요약, 발췌, 근거 조각을 구분하는 방향으로 설계합니다.

사용자가 확인하면 좋은 것

  • 외부 LLM provider를 선택했는지 확인합니다.
  • 회의 주제, 용어집, 문서 문맥에 민감한 내용이 있는지 확인합니다.
  • Confluence publish 전에는 위치와 제목을 확인합니다.
  • 민감한 회의라면 로컬 LLM 또는 요약/답변 비활성 흐름을 선택합니다.
