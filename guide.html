User Guide

회의를 남기고 다시 쓰는 법.

Minto2는 녹음 버튼 하나로 끝나는 앱이 아닙니다. 회의 전 문맥을 넣고, 회의 중 전사를 쌓고, 회의 후에는 요약·검색·내보내기로 업무 문서까지 이어갑니다.

사용법 빠른 시작 회의 전 준비 회의 중 기록 회의 후 정리 검색과 답변 내보내기

빠른 시작

처음에는 모든 설정을 만질 필요가 없습니다. 권한을 허용하고, 입력 소스를 고른 뒤, 회의를 시작하면 됩니다.

  1. 앱을 실행합니다.
    macOS 메뉴바에서 Minto2 상태를 확인하고 새 회의를 시작합니다.
  2. 마이크 권한을 허용합니다.
    시스템 사운드나 혼합 입력을 쓰는 경우 앱이 안내하는 추가 권한도 확인합니다.
  3. 입력 소스를 선택합니다.
    마이크, 시스템 사운드, 마이크+시스템 혼합 입력 중 회의 상황에 맞게 고릅니다.
  4. 회의를 종료합니다.
    종료 후 요약, 목차, 결정사항, 할 일, 미해결 질문, 전사를 확인합니다.

회의 전 준비

회의록 품질은 회의 전에 넣은 문맥의 영향을 받습니다. 필요한 만큼만 넣는 것이 좋습니다.

주제와 용어

회의 주제, 이번 회의에서 자주 나올 용어, 프로젝트명, 제품명, 사람 이름을 넣습니다. 전역 용어집이 있다면 관련 용어만 골라 씁니다.

참고 문서

로컬 md/txt/pdf, Notion, Confluence 문서를 붙일 수 있습니다. Minto2는 raw 문서 전체보다 요약·발췌·용어 중심으로 fail-soft하게 활용합니다.

문맥은 LLM에 대한 지시문이 아니라 참고자료입니다. 회의에 없는 내용을 만들지 않도록 전사와 문맥을 구분합니다.

회의 중 기록

회의 중에는 빠르게 보이는 preview와 저장에 쓰이는 final 전사가 분리됩니다. 마지막 발화 버퍼가 누락되지 않도록 종료 시점에 남은 구간을 정리합니다.

  • 마이크 입력은 내 목소리 중심 회의에 적합합니다.
  • 시스템 사운드는 온라인 회의 상대방 음성을 기록할 때 사용합니다.
  • 혼합 입력은 내 목소리와 상대방 음성을 함께 남길 때 사용합니다.
  • 화자 라벨은 회의 중 표시되고, 종료 후 더 안정적인 결과로 재정리될 수 있습니다.

회의 후 정리

회의가 끝나면 Minto2는 저장된 전사를 바탕으로 읽기 좋은 회의록을 만듭니다.

  • 요약과 목차로 긴 회의를 빠르게 훑습니다.
  • 결정사항, 할 일, 미해결 질문을 분리해 다음 행동으로 이어갑니다.
  • 문장 단위 화자 세그먼트로 누가 어떤 말을 했는지 확인합니다.
  • 화자 라벨은 사용자가 편집하거나 실명으로 지정할 수 있습니다.

교정 기능을 꺼도 원문 전사를 바탕으로 요약할 수 있습니다. 교정과 요약 설정은 독립적으로 다룹니다.

검색과 답변

회의록은 다시 찾으려고 남깁니다. 일반 검색은 LLM 없이도 동작하고, 답변 생성은 사용자가 요청했을 때만 실행됩니다.

  • 요약, 전사, 결정사항, 할 일, 미해결 질문을 검색합니다.
  • 임베딩 인덱스가 준비되면 의미가 가까운 회의 조각도 함께 찾습니다.
  • 답변 생성은 검색된 관련 근거 chunk만 LLM에 전달합니다.
  • 답변에는 출처 회의, 섹션, 시간 정보가 함께 표시되는 방향으로 설계되어 있습니다.

내보내기

정리한 회의록은 업무 도구로 옮길 수 있습니다.

  • Markdown 파일로 저장합니다.
  • 클립보드로 복사해 다른 도구에 붙여 넣습니다.
  • Confluence token이 있으면 위치와 제목을 확인한 뒤 publish합니다.
  • token이 없으면 Confluence 선택지는 비활성화하고 설정으로 안내합니다.
