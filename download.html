지금은 조심스럽게 배포합니다.

Minto2는 macOS 로컬 전사 앱입니다. 현재는 소스 빌드와 소수 공유용 자가서명 빌드를 중심으로 확인하고, 일반 사용자용 공증 배포는 별도 단계로 준비하는 전략이 가장 안전합니다.

현재 배포 상태

GitHub Releases 공개 빌드는 아직 없습니다.

현재 공개 저장소에는 바로 내려받을 정식 릴리스가 없습니다. 지금 단계에서 가장 현실적인 사용 방법은 직접 빌드하거나, 신뢰하는 소수 사용자에게 자가서명된 `.app` 번들을 공유하는 방식입니다.

일반 사용자에게 “다운로드 후 더블클릭으로 바로 실행”되는 경험을 제공하려면 Apple Developer Program, Developer ID 서명, 공증(notarization)이 필요합니다.

지원 환경

macOS

macOS 14 이상을 기준으로 합니다.

칩셋

Apple Silicon 환경을 기준으로 확인합니다.

첫 실행

STT 모델은 첫 실행 시 내려받을 수 있습니다.

직접 빌드해서 실행하기

개발 환경에서는 저장소를 받은 뒤 프로젝트 README의 개발자 안내를 따릅니다. Keychain 권한이 안정적으로 유지되도록 `swift run` 대신 프로젝트 스크립트를 사용합니다.

  1. 저장소를 받습니다.
    GitHub에서 코드를 내려받습니다.
  2. 자가서명 코드서명 인증서를 준비합니다.
    이름은 기본적으로 Minto2 Dev를 사용합니다.
  3. ./scripts/dev.sh run으로 실행합니다.
    빌드, 서명, 실행을 한 번에 처리합니다.

소수 공유용 빌드를 받았다면

자가서명 `.app` 번들은 공증된 앱처럼 조용히 열리지 않을 수 있습니다. 신뢰하는 경로에서 받은 빌드인지 먼저 확인하세요.

  1. 압축을 풉니다.
    `Minto2.app`을 원하는 위치로 옮깁니다.
  2. 처음에는 우클릭 후 열기를 사용합니다.
    macOS 경고가 뜨면 “열기”를 선택합니다.
  3. 필요한 권한을 허용합니다.
    마이크와 시스템 사운드 관련 권한을 허용합니다.

터미널에 익숙한 사용자는 quarantine 속성을 제거할 수 있지만, 공개 문서에서는 우클릭 후 열기를 기본 안내로 둡니다.

향후 일반 배포 전략

외부 일반 사용자에게 배포할 때는 아래 순서가 합리적입니다.

  1. .app 번들 생성 자동화
    릴리스 빌드, 아이콘, Info.plist, 리소스를 표준 앱 번들로 묶습니다.
  2. Developer ID 서명
    유료 Apple Developer Program으로 Developer ID Application 인증서를 사용합니다.
  3. 공증과 stapling
    Gatekeeper 경고를 줄이기 위해 notarytool 제출과 stapler 처리를 통과시킵니다.
  4. GitHub Releases 또는 DMG 배포
    사용자는 정식 릴리스 페이지에서 내려받고, 문서는 이 페이지에서 바로 연결합니다.
