Troubleshooting

안 될 때 먼저 볼 것들.

녹음, 전사, 요약, 검색, 내보내기는 각각 다른 권한과 설정을 사용합니다. 문제가 생기면 증상별로 경계를 나눠 확인하는 편이 빠릅니다.

문제 해결 권한 녹음과 전사 화자 라벨 교정·요약 검색·답변 내보내기
권한

마이크나 시스템 사운드가 잡히지 않을 때

  • macOS 시스템 설정에서 Minto2의 마이크 권한이 켜져 있는지 확인합니다.
  • 온라인 회의 소리를 기록하려면 시스템 사운드 입력 준비 상태를 확인합니다.
  • 권한을 바꾼 뒤에는 앱을 완전히 종료했다가 다시 실행합니다.

시스템 사운드와 마이크는 서로 다른 경로입니다. 내 목소리는 들어오는데 상대방 음성이 없거나, 반대로 상대방 음성만 들어오는 경우 입력 소스 선택을 먼저 봅니다.

전사

전사가 늦거나 일부 발화가 빠진 것처럼 보일 때

  • 회의 중 보이는 preview 전사는 임시 결과입니다. 저장 시 final 전사가 다시 정리됩니다.
  • 회의 종료 직후에는 남은 VAD 버퍼와 마지막 교정이 정리될 수 있습니다.
  • 소리가 너무 작거나 배경 소음이 크면 VAD가 발화 구간을 다르게 나눌 수 있습니다.
  • 파일 임포트는 먼저 16kHz mono PCM 변환을 거친 뒤 전사합니다.
화자

화자 라벨이 예상과 다를 때

  • 회의 중 라벨은 실시간 판단이고, 종료 후 VBx 기반 결과로 재정리될 수 있습니다.
  • 한 전사 청크 안에 여러 화자가 섞이면 저장 시 문장 단위로 다시 쪼갤 수 있습니다.
  • 화자분리 provider가 실패하면 채널 기반 라벨로 낮춰 계속 저장하는 fail-soft 경로가 있습니다.
  • 저장된 회의에서 화자 라벨을 편집하거나 실명으로 지정할 수 있습니다.
LLM

교정이나 요약이 만들어지지 않을 때

  • 선택한 provider가 교정 또는 요약 capability를 지원하는지 확인합니다.
  • API provider를 쓰는 경우 token이 Keychain에 저장되어 있는지 확인합니다.
  • 로컬 LLM을 쓰는 경우 모델 실행 상태와 응답 시간을 확인합니다.
  • 교정을 꺼도 원문 전사 기반 요약은 사용할 수 있습니다.

요약 실패가 저장이나 전사를 망치면 안 됩니다. Minto2는 실패하더라도 가능한 결과를 남기는 fail-soft 흐름을 우선합니다.

검색

검색 답변이 기대와 다를 때

  • 일반 검색은 LLM 없이 저장된 회의 텍스트를 찾습니다.
  • 답변 생성은 검색된 근거 chunk를 바탕으로 별도로 실행됩니다.
  • 질문이 너무 넓으면 회의명, 날짜, 결정사항, 할 일처럼 범위를 좁혀 입력합니다.
  • 답변에는 근거 회의와 시간이 함께 표시되는지 확인합니다.
내보내기

Confluence 내보내기가 비활성화될 때

  • Confluence token이 없으면 선택지가 비활성화됩니다.
  • 설정에서 token을 등록한 뒤 다시 내보내기를 시도합니다.
  • publish 전에는 위치와 제목을 확인합니다.
  • 외부 도구 연동이 어렵다면 Markdown 파일 또는 클립보드 복사를 fallback으로 사용합니다.
