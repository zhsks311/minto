데이터 경계

로컬이라고 뭉뚱그리지 않습니다.

음성, 텍스트, LLM 요청은 서로 다른 위험을 갖습니다. Minto2는 사용자가 이해할 수 있도록 경계를 나눕니다.

안

음성 전사는 기기 안에서

마이크·시스템 사운드·혼합 입력을 Mac에서 처리합니다. STT 모델은 내려받지만, 회의 음성 자체는 전사를 위해 서버로 올라가지 않습니다.

음성은 안 나감
밖

AI 정리는 선택했을 때만

교정·요약·검색 답변에 클라우드 LLM을 고르면 텍스트가 나갈 수 있습니다. 검색 답변은 전체 회의가 아니라 관련 근거 조각을 보냅니다.

사용자 선택
완

끝까지 로컬도 가능

로컬 LLM을 선택하면 교정과 요약까지 Mac 안에서 끝낼 수 있습니다. 토큰·프롬프트·전사 원문은 로그에 남기지 않는 것을 원칙으로 둡니다.

전 과정 로컬 가능
회의 전 · 중 · 후

기록은 회의가 시작되기 전에 이미 좋아집니다.

좋은 회의록은 녹음 버튼만으로 나오지 않습니다. Minto2는 회의 문맥을 먼저 받고, 회의 중에는 놓치지 않고, 끝난 뒤에는 다시 쓰기 쉽게 정리합니다.

회의 전

  • 회의 주제와 이번 회의 용어를 적습니다.
  • 전역 용어집에서 관련 용어만 고릅니다.
  • 로컬 파일, Notion, Confluence 문서를 참고자료로 붙입니다.

회의 중

  • 마이크와 시스템 오디오를 함께 전사합니다.
  • VAD로 발화 구간을 나누고 preview/final 전사를 구분합니다.
  • 실시간 화자 라벨을 보여주고 실패하면 채널 기반 라벨로 낮춥니다.

회의 후

  • 요약, 목차, 결정사항, 할 일, 미해결 질문을 만듭니다.
  • 단어 타임스탬프로 문장 단위 화자 세그먼트를 정리합니다.
  • 검색·답변·Markdown·클립보드·Confluence 내보내기로 이어집니다.
현재 기능

회의 앱이 아니라, 회의 지식 도구.

2026년 6월 29일 기능 명세 기준으로 현재 구현된 흐름을 중심으로 정리했습니다.

🎙

실시간 전사

마이크, 시스템 사운드, 혼합 입력을 받아 회의 목록의 현재 회의에 바로 쌓습니다.

🧩

preview / final

회의 중 미리보기와 확정 전사를 분리해 마지막 VAD 버퍼까지 놓치지 않도록 설계했습니다.

🗣

화자 라벨

실시간 라벨과 종료 후 VBx 결과를 이용해 문장 단위 화자 세그먼트로 정리합니다.

📂

파일 임포트

음성·영상 파일을 16kHz mono PCM으로 변환해 같은 전사·요약 흐름으로 처리합니다.

📖

용어집

전역 용어집과 회의별 용어를 함께 써서 고유명사와 전문용어를 더 안정적으로 다룹니다.

📎

문서 문맥

md, txt, pdf, Notion, Confluence 문서를 회의 시작 참고자료로 붙일 수 있습니다.

🔎

검색과 답변

요약·전사·결정사항·할 일·미해결 질문을 검색하고, 답변에는 출처와 시간이 붙습니다.

🧠

LLM 선택

로컬 LLM, OpenAI, Gemini, Claude, OpenRouter, 로그인 계정, Claude Code CLI 경로를 고를 수 있습니다.

📤

내보내기

Markdown 파일, 클립보드, Confluence로 보냅니다. 토큰이 없으면 설정으로 안내합니다.

🔐

Keychain 저장

비밀값은 기본적으로 Keychain에 저장하고, 개발 환경에서만 file 기반 저장을 선택할 수 있습니다.

🧰

메뉴바 앱

회의 시작과 종료를 macOS 메뉴바에서 빠르게 제어합니다.

🧪

STT 벤치마크

엔진 선택은 감이 아니라 CER 기준 벤치마크로 비교할 수 있게 도구를 둡니다.

검색 결과가 아니라, 근거가 있는 답을 봅니다.

회의록은 나중에 다시 쓰려고 남깁니다. Minto2의 답변은 저장된 회의에서 찾은 근거 조각을 바탕으로 만들고, 회의·섹션·시간을 함께 보여주는 방향으로 설계되어 있습니다.

일반 검색LLM 없이도 요약·전사·결정사항·할 일·미해결 질문을 찾습니다.
Semantic search임베딩 인덱스가 준비되면 문장 의미가 가까운 회의 조각까지 함께 찾습니다.
답변 생성사용자가 요청했을 때만 관련 근거 chunk를 LLM에 전달합니다.
공유Markdown, 클립보드, Confluence로 회의 내용을 옮길 수 있습니다.

회의가 끝난 뒤에도 쓸 수 있게.

코드와 빌드 방법은 GitHub에 있습니다. macOS 14 이상, Apple Silicon 환경에서 직접 확인해 보세요.
